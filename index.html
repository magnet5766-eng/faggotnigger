Core features

Built tight.
Stays tight.

Every component is chosen with intention. Nothing runs on your system that you didn't put there.

Immutable root filesystem

The root filesystem ships read-only and verified via dm-verity on every boot. Malware can't persist across reboots. System state stays exactly as you set it, always.

Zero telemetry

No crash reports. No usage analytics. Nothing leaves your machine unless you explicitly send it.

Verified boot chain

Secure Boot integration with a full chain-of-trust from firmware to userspace. Tampering is detected before the OS loads.

Container-first apps

Applications run isolated in OCI containers. One misbehaving app can't touch the rest of your system.

Network firewall by default

Outbound connections are denied by default. Every allowed connection is an explicit choice, not an accident.

Atomic updates

Updates apply to a new snapshot, not your running system. Roll back to any prior state in seconds.

Philosophy

Privacy is a
default, not a feature.

Most operating systems treat privacy as a settings menu. Fortis treats it as a constraint that everything else is built around.

We don't collect data we don't need. We don't enable services you didn't ask for. We don't ship software that phones home.

  • Principle of least privilege applied everywhere
  • No background services without explicit consent
  • Full source available, audit it yourself
  • Reproducible builds, verified by the community
How it works

From boot to desktop, secured.

Fortis chains every layer of your system together. Compromise one, and the chain breaks visibly.

01

Verified firmware

UEFI Secure Boot validates the bootloader signature before handing off control.

02

Signed kernel

The Fortis kernel is signed and verified. Unsigned modules are rejected at load time.

03

dm-verity root

The root filesystem is hash-verified block by block. Any corruption halts the boot.

04

Isolated userspace

Apps run in containers with explicit capability grants. The default answer is no.
