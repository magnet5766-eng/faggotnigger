Installation

Download the ISO from the downloads page and verify the signature before proceeding. Fortis supports UEFI boot only. Legacy BIOS is not supported.

Write the ISO to a USB drive of at least 4GB. On Linux:

dd if=fortis-1.0.0-x86_64.iso of=/dev/sdX bs=4M status=progress oflag=sync

Replace /dev/sdX with your actual USB device. Boot from the USB, select your target drive, and the installer handles the rest.

Verifying the ISO

Import the Fortis GPG key, then verify the detached signature alongside the ISO:

gpg --import fortis-release.asc
gpg --verify fortis-1.0.0-x86_64.iso.sig fortis-1.0.0-x86_64.iso

You should see Good signature from "Fortis Release Key". If you see anything else, do not boot the image.

First boot

On first boot, Fortis generates your machine ID, sets up the dm-verity hash tree, and seals your TPM PCR measurements. This takes roughly 90 seconds. Do not interrupt it.

After setup completes, you will be prompted to create a user account. Root login is disabled by default and cannot be re-enabled through the standard interface.

Immutable root filesystem

The root partition is mounted read-only via dm-verity. Writes intended for system paths are transparently redirected to an overlay layer. On next boot, the overlay is discarded unless you explicitly commit changes via:

fortis commit --message "Added ffmpeg"

This snapshots your current overlay into a new root generation. You can roll back to any prior generation at boot time from the GRUB menu, or via:

fortis rollback --generation 3

Running applications

Applications are distributed as OCI container images. Install and run them with the fortis app subcommand:

fortis app install org.mozilla.firefox
fortis app run org.mozilla.firefox

Each app runs with a minimal capability set. Network access, filesystem access, and device access are all opt-in per application, declared in the image manifest and visible to you before install.

Firewall defaults

Outbound connections are denied by default for all non-system processes. To allow an app to reach the network, add a rule:

fortis firewall allow org.mozilla.firefox --egress tcp:443 --egress tcp:80

Rules are stored in /etc/fortis/firewall.d/ and applied on boot. You can audit all active rules with fortis firewall list.
